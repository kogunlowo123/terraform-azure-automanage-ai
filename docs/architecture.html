Azure Automanage

Executive Summary

Azure Automanage module with AI-driven infrastructure optimization providing a complete VM management platform. Provisions Automanage configuration with antimalware, automated backup via Recovery Services Vault, maintenance windows for OS patching (Linux and Windows), Azure Policy for compliance enforcement (custom and built-in), metric alerts for CPU/memory/disk/availability, Action Groups for email notifications, an Automation Account with PowerShell runbooks for self-healing remediation, and centralized Log Analytics with diagnostic settings across all components.

Data Flow

1
VMs are registered with Automanage configuration for antimalware and automated management.
2
Maintenance configuration assigns weekly patching windows for Critical and Security patches.
3
Recovery Services Vault protects VMs with scheduled backups and multi-tier retention.
4
Azure Policy enforces compliance rules (backup enabled, monitoring agent installed).
5
Metric alerts trigger when thresholds are breached (CPU, memory, disk, availability).
6
Action group sends email notifications to operations team.
7
Automation runbooks execute self-healing actions (restart unhealthy VMs).
8
All diagnostic data flows to Log Analytics workspace for centralized analysis.

Deployment

# Register resource providers
az provider register --namespace Microsoft.Automanage
az provider register --namespace Microsoft.RecoveryServices
az provider register --namespace Microsoft.Automation

# Deploy
terraform init
terraform plan -out=tfplan
terraform apply tfplan

# Verify backup protection
az backup item list \
  --resource-group myapp-rg \
  --vault-name myapp-recovery-vault
